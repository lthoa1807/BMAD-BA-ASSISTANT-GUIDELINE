# Step 2: Outline And Sections

## Goal

Design the template structure and section file list.

## Instructions

Create a section outline appropriate to the document type. For each section, define:

- Section filename
- Purpose
- Required source inputs
- Expected content
- Evidence or citation guidance when relevant
- Assumptions or open questions that affect writing
- Review owner or reviewer type when known

For large documents, prefer numbered section files such as `01-overview.md`, `02-scope.md`, and `03-process.md`.

## Output Fragment

Create the template outline and section file list.

## Step Documentation

Before moving to the next step, create or update `{ba_artifacts}/documents/<document-name>/template/step-02-outline-and-sections-notes.md` with:

- Workflow and step name
- Inputs reviewed, including `step-01-template-inputs-notes.md` if available
- Questions asked
- User responses and evidence
- Section outline with `DOC-###` IDs where useful
- Decisions with `DEC-###` IDs
- Assumptions
- Risks with `RISK-###` IDs
- Open questions with `Q-###` IDs
- Output fragment
- Next action

Update `{ba_artifacts}/ba-worklog-index.md` with the completed step, artifact path, key decisions, risks, open questions, and next action.
