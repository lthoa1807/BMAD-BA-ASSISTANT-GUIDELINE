# Step 1: Template Inputs

## Goal

Confirm the inputs needed to generate a useful document template.

## Instructions

Review the document plan if available. Confirm:

- Document type and working title
- Audience and intended use
- Required sections and optional sections
- Source inputs and evidence approach
- Required review checkpoints
- Output expectations for large documents, including whether each section should be a separate Markdown file

If no document plan exists, ask the minimum questions needed to create a compact template input summary.

## Output Fragment

Create a template input summary with document purpose, audience, section needs, evidence needs, and unresolved questions.

## Step Documentation

Before moving to the next step, create or update `{ba_artifacts}/documents/<document-name>/template/step-01-template-inputs-notes.md` with:

- Workflow and step name
- Inputs reviewed
- Questions asked
- User responses and evidence
- Template input findings with `DOC-###` IDs where useful
- Decisions with `DEC-###` IDs
- Assumptions
- Risks with `RISK-###` IDs
- Open questions with `Q-###` IDs
- Output fragment
- Next action

Update `{ba_artifacts}/ba-worklog-index.md` with the completed step, artifact path, key decisions, risks, open questions, and next action.
