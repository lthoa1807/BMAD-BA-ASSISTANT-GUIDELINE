# Short Document Writing Rules

## Voice And Style

- Write for the target reader, not for the implementation team.
- Use direct action verbs when the document includes instructions.
- Prefer short sections and clear topic names.
- Keep terminology consistent with the product and business process.
- Define acronyms on first use unless they are universally known to the audience.

## Evidence And Accuracy

- Only state facts, product behavior, or process rules supported by the source document, user confirmation, or accepted assumptions.
- Mark uncertain facts or behavior as open questions instead of confirmed statements.
- Preserve important warnings, prerequisites, and exceptions from the source.
- Do not invent screenshots, menu names, field labels, or system responses.

## Images

- Place images near the related topic, task, or explanation.
- Use concise alt text that explains the purpose of the image.
- If image placement is uncertain, keep the image and record the uncertainty in review notes.
- Do not drop extracted images without documenting the reason.

## Formatting

- Use Markdown headings in numeric order.
- Use numbered lists for ordered steps.
- Use bullet lists for unordered notes, prerequisites, and cautions.
- Use tables for field definitions, status meanings, and change history.
- Keep final Markdown as the source of record for Word export.
