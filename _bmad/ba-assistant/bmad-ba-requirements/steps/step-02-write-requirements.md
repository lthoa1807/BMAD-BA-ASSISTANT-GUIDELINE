# Step 2: Write Requirements

## Goal

Draft clear, testable requirements across functional and cross-functional concerns.

## Instructions

For each requirement, include:

- Requirement ID
- Process area
- Business objective
- Requirement statement
- Priority
- Fit-gap reference
- Acceptance criteria
- Data considerations
- Integration considerations
- Reporting considerations
- Security and control considerations
- Migration or cutover considerations
- Dependencies
- Assumptions and open questions

Use business language first. Add solution notes only when supported by a fit-gap decision.

## Output Fragment

Create the requirements list with acceptance criteria.
