# Step 1: Structure

## Goal

Choose the requirements structure and confirm the source inputs.

## Instructions

Use available discovery and fit-gap artifacts. If missing, ask for the minimum context needed.

Define the structure by process area, scenario, epic, module, product area, or implementation wave. Capture source inputs:

- Discovery brief
- Fit-gap matrix
- Process maps
- Policy and control documents
- Data objects and migration notes
- Integration and reporting inventory
- Security roles and approval rules
- Open decisions and assumptions

## Output Fragment

Create a requirement structure and source-input summary.

## Step Documentation

Before moving to the next step, create or update `{ba_artifacts}/requirements/step-01-structure-notes.md` with:

- Workflow and step name
- Inputs reviewed, including discovery and fit-gap artifacts when available
- Questions asked
- User responses and evidence
- Requirement structure findings with draft `REQ-###` ranges where useful
- Upstream `DISC-###` and `GAP-###` links where available
- Decisions with `DEC-###` IDs
- Assumptions
- Risks with `RISK-###` IDs
- Open questions with `Q-###` IDs
- Output fragment
- Next action

Update `{ba_artifacts}/ba-worklog-index.md` with the completed step, artifact path, key decisions, risks, open questions, and next action.
