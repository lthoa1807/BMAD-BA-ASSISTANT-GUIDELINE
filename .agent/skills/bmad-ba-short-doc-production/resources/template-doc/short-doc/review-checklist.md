# Short Document Review Checklist

## Source Coverage

- All relevant source sections were reviewed.
- Important source tables, lists, screenshots, and warnings are represented.
- Excluded source content is listed with a reason.
- Unsupported or unreadable source content is documented.

## Structure

- The document has a clear overview, scope, key context, main content, relevant actions or details, references, and change history.
- Headings are ordered and meaningful.
- The document is readable for the intended audience.

## Accuracy

- Facts and instructions are supported by the source or user confirmation.
- Assumptions are listed separately from confirmed facts.
- Open questions are clear and actionable.
- Accepted caveats are visible in the final summary or changelog.

## Images

- Every extracted image is used or documented as omitted.
- Image links work from the final Markdown location.
- Image placement is reasonable or flagged for review.
- Word export includes expected images when `.docx` is produced.

## Final Package

- Source, template, working records, images, final outputs, and manifest are present.
- Package omissions are documented.
- Cleanup was not performed before package verification and user confirmation.
