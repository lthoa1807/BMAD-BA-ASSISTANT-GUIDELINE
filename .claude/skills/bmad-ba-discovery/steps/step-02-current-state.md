# Step 2: Current State

## Goal

Capture how work is done today and where change is expected to create value or risk.

## Instructions

Build a current-state inventory covering:

- End-to-end process flows and handoffs
- Source systems, spreadsheets, forms, reports, and shadow tools
- Master data and transactional data objects
- Approval rules, segregation of duties, audit controls, and compliance constraints
- Pain points, workarounds, manual effort, rework, and exception volume
- Integrations, reporting, analytics, and migration dependencies
- Variants by site, country, entity, customer group, product type, or channel

Challenge vague statements by asking for examples, frequency, volume, and business impact.

## Output Fragment

Create a current-state summary with process inventory, evidence, pain points, controls, data, reporting, and integration notes.
